# 论文表述风格硬约束（长期生效）

## 背景

在论文正文中出现“写作说明式元叙述”会削弱顶会/顶刊文本的技术密度与专业性。

## 硬约束

1. 禁止使用“为符合可证明安全论文的写作口径”“为便于叙述”“下文将介绍”等写作过程导向句式。
2. 所有段落应直接陈述技术对象：系统模型、威胁假设、安全定义、构造流程、证明结论、实验发现。
3. 安全分析默认采用形式化语言：PPT 对手、泄露函数、可区分优势、`negl(\lambda)`。
4. 润色流程中若发现元叙述句，先删后改，不做轻微修辞替换。
## 领域用语禁止列表（2026-02-23 新增）

以下用语不得出现在密码学论文正文中：

### 软件工程用语
- "工程可用性"、"工程可落地性"、"接口"、"解耦"、"数据平面"、"在实现层面"
- 上述术语仅适用于系统/软件工程文献，不得用于描述密码学构造或安全性质

### 政策/营销语言
- "数字经济核心基础设施"、"合规治理"、"信任度与采纳意愿"、"数据要素流通"
- 此类表述属于政策宣传或市场营销，不得出现在学术论文技术段落

### 教科书式枚举
- 不得以"从…到…再到…"的递进句式做面面俱到的文献综述导语
- 文献综述应直接按技术维度（如：静态/动态、单用户/多用户、可验证/不可验证）组织

### 术语滥用
- "可忽略"（negligible）仅限指 $\mathsf{negl}(\lambda)$，不得描述假阳性率、性能开销等
- 避免将运维故障（存储损坏、配置错误）纳入密码学威胁模型

## 行文风格约束（2026-02-23 新增）

1. 避免短句堆砌：每个算法或协议步骤的描述应包含完整的输入-操作-输出语义。
2. 算法签名应标注输出类型（如 `Setup(1^λ) → (K, st, EDB)`），使形式定义自洽。
3. 图表后的正文段落不应复述图表已展示的信息，而应阐述图表对后续内容的承载关系或设计启示。
4. 符号表采用统一平铺格式，无需按类别分组标注。
5. 密码学原语在"符号约定"节仅给出类型签名与安全性要求，不讨论原语与具体方案的绑定关系。
6. 协议交互流程节应采用通用 SSE 语言描述，具体方案的令牌结构仅在该方案的专属小节中展开。
7. 正文不宜过多使用分条叙述（enumerate/itemize），应以连贯、逻辑清晰的段落形式呈现内容。引言与讨论部分必须采用完整段落展现严密推理与深入分析；方法与结果部分可适度使用分条突出关键步骤或数据，但应控制数量，避免碎片化。分条仅在列举三项及以上并列要点或协议步骤时使用。

## 常见违规模式与修正经验（2026-02-25 新增）

### 1. 分条叙述违规（第15条）

**典型违规**：
```latex
本文研究意义体现在两个维度。
\begin{enumerate}
    \item 理论意义：...
    \item 应用价值：...
\end{enumerate}
```

**修正策略**：
- 2项以下内容必须改为连贯段落
- 使用"在X层面，...在Y层面，..."的段落结构
- 保持逻辑递进但避免碎片化

**修正示例**：
```latex
本文研究具有以下意义。在理论层面，...在应用层面，...
```

### 2. 图表元叙述违规

**典型违规**：
- "图X对本文所关注的...进行了统一抽象"
- "由表X可见"
- "图X展示了本文从...到...的整体研究路径"

**修正策略**：
- 删除"对...进行了抽象/展示/说明"等元叙述
- 直接陈述技术内容，图表编号作为括号补充
- 或简化为"(图X)"放在句末

**修正示例**：
- 原文："图1对本文所关注的系统角色进行了统一抽象。该场景涉及三类实体..."
- 修正："该场景涉及三类实体...（图1）"

### 3. 模板化 AI 过渡句式

**典型违规**：
- "更为关键的是"
- "值得注意的是"
- "其研究意义体现在...维度"
- "可归纳为X个方向"

**修正策略**：
- 直接删除过渡词，让技术内容自然衔接
- 将"体现在...维度"改为直接陈述
- 将"可归纳为X个方向"改为"按X维度梳理"

### 4. 写作导向表述

**典型违规**：
- "需要关注的核心结构是"
- "后续可验证机制可在..."
- "下文将介绍"

**修正策略**：
- 将"需要关注"改为"核心结构为"
- 删除"后续/下文"等时序导向词
- 用现在时直接陈述技术事实

### 5. 符号一致性检查要点

**常见问题**：
- 同一概念使用多套记号（如 $\mathcal{W}_i$ vs $w_i$）
- 符号表与正文不一致

**检查流程**：
1. 首次定义时明确符号含义
2. 全文搜索该符号的所有出现
3. 确保符号表与正文完全一致
