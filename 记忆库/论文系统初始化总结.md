## 二、X 光解构（用于后续统一口径）

### 1. 核心痛点

- 现有方案多关注泄露控制，较少约束“服务器是否按协议诚实返回结果”
- 资格检验与结果返回两个环节都存在可被操控的空间

### 2. 解题机制

- 在资格检验环节，用 MHT 让每次判定携带可验证证据
- 在结果返回环节，把承诺信息嵌入既有索引，避免独立验证索引开销

### 3. 创新增量

- 从“概率过滤”升级为“可证明资格检验”
- 从“独立验证结构”转向“索引同构嵌入式验证”

### 4. 关键假设与边界

- 授权管理方可信
- 默认不考虑撤权用户与服务器串谋的强对抗
- 目标是“尽量不扩大”原有 SSE 泄露函数，而非零泄露

### 5. 逻辑主线（ASCII）

```text
Problem
  |
  +--> Eligibility Step Unverifiable (Bloom-style)
  |         |
  |         +--> MHT-based Verifiable Eligibility
  |
  +--> Result Return Unverifiable
            |
            +--> Embedded Commitment Verification
                      |
                      +--> Detect omission/forgery with bounded overhead
```
