本轮工作聚焦 `HUST-PhD-Thesis-Latex/body/chapter/bf.tex` 的第二章关键过渡：在保留 Nomos 基线语义的前提下，完成以下任务：

2. 重写 `2.2 Nomos协议` 叙述，统一为密码学论文口径。
3. 显式补入 Nomos 的 RBF 创新点与参数机制（$\ell,k,\beta_i$）。
4. 形成从 Nomos 到 “基于 MHT 的可验证资格检验” 的动机过渡链路。

## 二、已完成改造（章节级）

### 1) 算法块层

- 完成算法 `\mathsf{Setup}`、`\mathsf{Update}`、`\mathsf{GenToken}`、`\mathsf{Search}` 的重构与符号统一。
- `GenToken` 明确为 `Client -> Gate-keeper -> Client` 三段流程。
- `Search` 重写为 `Client -> Server -> Client`，并与图示变量对齐。
- 统一了 `\overline{\mathsf{bxtrap}}` 等关键记号，减少算法-正文记号漂移。

### 2) 协议叙述层（2.2）

- 移除“工程化基线/写路径/查路径”等非密码学写法。
- 改为标准术语链：初始化、更新、令牌生成、检索、验证。
- 补入 RBF-backed `\mathsf{XSet}` 的作用描述与参数语义，明确：
  - 更新侧按 $\ell$ 写入 `\{\mathsf{xtag}_i\}`；
  - 令牌侧按 $k$ 采样 `\{\beta_i\}`；
  - 检索侧据此执行成员测试。

### 3) 动机过渡层

- 在“设计动机与问题分析”节首加入过渡段：
  - 先确认 Nomos 已解决的问题（隐私 + 授权链路）；
  - 再指出可验证性缺口（执行不可核验、可偏离）；
  - 最后引出 MHT 改造方向（证据绑定 + 状态一致性）。

## 三、关键写作结论（可复用）

1. 当论文是“基于 A 改进 B”时，必须有显式过渡段：  
`A 的能力边界 -> 不足与攻击面 -> B 的设计目标`。

2. RBF/BF 相关文字至少覆盖三层信息：
- 协议映射层（变量与算法关系）；
- 统计性质层（假阳性及参数）；
- 可验证张力层（为何高效但不可直接验证）。

3. 对密码学章节，优先使用“性质/目标 -> 机制 -> 结论”的行文结构，避免流程导向或工程口语化词汇。
