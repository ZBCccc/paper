# VQNomos 方案命名设计

**日期**：2026-03-08
**状态**：已批准

## 背景

硕士论文需要为基于 Nomos 协议的可验证扩展方案设计一个专业、简洁的学术名称。

## 方案特点

1. **基础协议**：Nomos（多用户多关键词动态 SSE）
2. **第一层扩展**（第二章）：基于 Merkle Hash Tree 的可验证资格检验机制 $\Pi_{\mathsf{VQ}}$（QTree）
3. **第二层扩展**（第三章）：基于嵌入式承诺的地址绑定验证机制
4. **核心价值**：将资格判定从布尔输出提升为可独立验证的密码学证明

## 命名要求

- 学术规范性
- 易记性
- 与 Nomos 的关联性
- 体现"可验证"核心特性

## 候选方案

### 方案一：VQNomos（推荐）

- **VQ** = Verifiable Qualification（可验证资格）
- 直接对应第二章的 $\Pi_{\mathsf{VQ}}$ 机制
- 保留 Nomos 基础，清晰表明扩展关系
- 简洁（7个字母），易记易读
- 符合密码学领域前缀式命名惯例（V-ORAM, V-SSE）

### 方案二：Nomos-VC

- **VC** = Verifiable Commitment（可验证承诺）
- 后缀式命名，与 Nomos 关联紧密
- 涵盖两层验证机制
- 符合协议命名惯例（TLS-PSK, AES-GCM）

### 方案三：Nomos+

- 最简洁（6个字符）
- "+" 符号暗示增强版本
- 缺点：未明确体现"可验证"特性

## 最终决策

**选定方案**：**VQNomos**

**理由**：
1. 技术一致性：直接对应论文核心机制 $\Pi_{\mathsf{VQ}}$
2. 平衡性：在简洁性和描述性之间取得最佳平衡
3. 学术规范：符合密码学领域命名惯例
4. 可引用性：易于在论文中引用和解释

## 后续行动

- 在实验章节中使用 VQNomos 作为方案标识
- 在对比实验中使用：Nomos（基线）、MC-ODXT（对比方案）、VQNomos（本文方案）
