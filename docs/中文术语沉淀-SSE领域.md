## 一、核心概念术语

|---|---|---|---|

| trapdoor | **陷门** | ~~搜索令牌~~（功能不同，陷门特指密钥衍生的搜索输入） | 刘文心 |
| search token | **搜索令牌** | ——（与陷门区分：令牌在多用户场景下经 OPRF 盲化） | 陈天阳 |
| encrypted database (EDB) | **加密数据库** | 密文数据库 | 陈天阳 |
| leakage function | **泄露函数** | 泄漏函数（"漏"/"露"均见，"泄露"更正式） | 刘文心、陈天阳 |
| access pattern | **访问模式** | ——  | 王贇玲、刘文心 |
| search pattern | **检索模式** / **搜索模式** | ——（两种均在综述中出现，同一文中保持一致） | 王贇玲 |
| update pattern | **更新模式** | —— | 陈天阳 |
| forward privacy / forward security | **前向隐私** | ~~前向安全~~（密码学语境中"安全"指 computational security，"隐私"对应 privacy；SSE 领域标准用"前向隐私"） | 陈天阳第1.2.3节 |
| backward privacy / backward security | **后向隐私** | ~~后向安全~~ | 陈天阳第1.2.4节 |
| file identifier | **文件标识符** | ~~文档标识符~~（SSE 领域以"文件"为主，数据库记录场景用"记录"） | 陈天阳全文 |
| inverted index | **倒排索引** | ——  | 王贇玲 |
| conjunctive keyword search | **合取关键词检索** | ——  | 王贇玲 |

| false positive | **假阳性** | 误判（假阳性更精确） | 王贇玲 |
| query complexity | **检索复杂度** | 查询复杂度（密文检索场景用"检索"） | 陈天阳 |
| ciphertext update / record update | **密文更新** | —— | 陈天阳 |
| oblivious RAM (ORAM) | **不经意随机访问机** | 遗忘 RAM | 陈天阳 |

## 二、安全性质与模型术语

| 英文 | **规范中文** | 说明 |
|---|---|---|
| adaptive security | **自适应安全** | 陈天阳"适应性安全"也见于早期文献，建议统一 |
| non-adaptive security | **非自适应安全** | —— |
| L-adaptive security | **L-自适应安全** | 刘文心 |
| semantic security (SS) | **语义安全** | 刘文心第4节 |
| indistinguishability security (IND) | **不可区分性安全** | 刘文心 |
| soundness | **健全性** | ~~正确性~~（Soundness ≠ Correctness，前者关于恶意方，后者关于诚实方） |
| completeness | **完整性** | 协议语境（区分"结果完整性"与"安全证明步骤"） |
| freshness | **新鲜性** | 在协议安全属性语境下可用，但用于描述地址/计数器时应改为**唯一性** |
| binding | **绑定性** | 承诺方案专用属性 |
| hiding | **隐藏性** | 承诺方案专用属性 |
| EUF-CMA | **存在性不可伪造性（自适应选择消息攻击）** | 签名安全属性 |

## 三、工程表达 → 学术表达对照

| 工程化/口语化 | **学术规范替换** | 备注 |
|---|---|---|
| 基线方案 | **Nomos 原始方案** / **Nomos 方案** | "基线"（baseline）是工程测试用语，学术对比用"原始方案"或直接称方案名 |
| 基线 | **Nomos 方案** / **原始 Nomos 构造** | 在实验章节比较开销时，可用"以 Nomos 方案为对比基准" |
| 基准（benchmark） | **对比基准** | 实验章节可用 |
| 基线层 | **基础层** / **Nomos 方案层** | 技术路线图标注 |
| 地址的新鲜性（描述计数器递增语义） | **地址的唯一性** | "新鲜性"是安全属性名，描述计数器语义时应用"唯一性" |
| 写路径 / 查路径 | **更新条目** / **检索条目** | 约束10禁用工程表达 |
| 查询流水 | **查询历史** | —— |
| 高效 / 安全（修饰词，无量化） | **需附带复杂度界或安全参数** | 约束5 |

## 五、字词风格参考（从参考论文中提炼）

### 5.1 动词用词规范

| 操作 | **规范用词** | 避免使用 |
|---|---|---|
| 服务器执行搜索 | **执行检索** | 执行搜索（可接受）/ 进行搜索 |
| 用户发起请求 | **发起检索请求** / **提交陷门** | 发送请求 |
| 写入索引 | **写入** / **添加记录** | 写入条目 |
| 删除记录 | **删除记录** | 删除文件 |
| 输出结果 | **返回** / **输出结果集** | 给出结果 |

### 5.2 参考论文标准句式

- 「本章首先…，然后…，最后…」→ 禁用元叙述（约束1）
- 「[方案] 实现了 [性质 $P$]，在 [安全模型] 下给出了 [归约证明]」✅
- 「[方案] 基于 [假设] 构造，其检索复杂度为 $O(\cdot)$，通信开销为 $O(\cdot)$」✅
- 「在 PRF 安全性与 OPRF 安全性假设下，算法 X 满足前向隐私」✅ （参考陈天阳命题格式）
