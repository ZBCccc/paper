# PDF 转 Markdown 流程（论文系统）

## 目标

将 `ref-thesis/` 中的 PDF 文献抽取为可检索的 Markdown 文稿，统一归档到：

`01-论文生产/素材库/论文Markdown库/`

默认抽取方式：`ljg-xray-paper`（结构化解构），而不是简单全文格式转换。

## 步骤

1. 使用 `ghostscript` 抽取文本：

```bash
gs -q -dNOPAUSE -dBATCH -sDEVICE=txtwrite -sOutputFile=/tmp/paper_raw.txt "<PDF路径>"
```

2. 采用 Xray 抽取框架（核心步骤）：

- 去噪：跳过冗长背景与模板化表述，聚焦关键贡献
- 提取：核心痛点、关键机制、创新增量
- 批判：隐形假设与未解问题
- 产出：ASCII 逻辑图 + 餐巾纸公式/图

3. 写入 Markdown 主文件：

- 文件名：`作者-年份-标题关键词.md`
- 在文档头部记录来源路径、DOI、转换时间
- 结构建议：
  - 【1. 核心痛点】
  - 【2. 解题机制】
  - 【3. 创新增量】
  - 【4. 批判性边界】
  - 【5. 一言以蔽之】
  - 逻辑结构图（ASCII）

4. 可选：保留 raw 备份

- 文件名：`作者-年份-标题关键词.raw.md`
- 用途：仅用于回溯原文抽取，不作为主阅读文档

5. 更新系统文件（至少其一）：

- `控制面板.md`（导航）
- `记忆库/`（若需要记录该文贡献）
- 对应素材库条目（可选）

## 质量提示

双栏论文自动抽取可能出现：

1. 断词（如 `construc- tion`）
2. 列间拼接
3. 少量符号丢失

若用于精确引用，仍需回看原 PDF；Xray 文档用于“快速理解与写作决策”，不是证据替代品。
