# 工作流程

## 论文写作主流程

1. **选题收集**：在 `01-论文生产/选题管理/00-选题记录.md` 记录研究想法
2. **待深化管理**：将具备可行性的方向移动到 `01-论文生产/选题管理/01-待深化/`
3. **待投稿管理**：阶段性完成稿件移至 `01-论文生产/选题管理/02-待投稿/`
4. **投稿归档**：投稿后统一归档至 `01-论文生产/选题管理/03-已投稿/`

## 论文工程协作流程（当前核心）

1. 在 `HUST-PhD-Thesis-Latex/body/chapter/` 推进章节正文
2. 把结构性结论回写到 `docs/` 对应文件
3. 把关键研究记忆沉淀到 `docs/memory/`
4. 在 `rules/控制面板.md` 维护章节状态与近期任务

## 参考资料管理

- **核心论文库**：记录定义性与奠基性论文
- **相关工作库**：按技术路线梳理进展与差异
- **基础库**：维护术语、模型、原语、假设等底层知识
- **论文 Markdown 库**：将 `ref-thesis/` 中 PDF 抽取为 Xray 结构化 `.md`（默认 `ljg-xray-paper`），统一存放在 `docs/论文Markdown库/`

## PDF 文献处理规则

1. 新增 PDF 后，优先按 `ljg-xray-paper` 抽取为 Xray Markdown
2. Markdown 文件统一放入 `docs/论文Markdown库/`
3. 文件名使用 `作者-年份-标题关键词.md` 规范
4. 抽取后在文档头部记录来源 PDF 路径与 DOI
5. 需要保留全文转储时，另存为 `*.raw.md`，不作为主阅读材料

## 与 AI 交互规范

1. 每次开工先说明你在写哪一章、卡在哪一段
2. 明确输出类型：改写、补证、补实验、查引用、结构优化
3. 让 AI 同步更新对应系统文件，而不仅是单次回答
4. 完成后回写进度到 `rules/控制面板.md` 或 `docs/memory/`

## 日常执行节奏

### 每日（10-20 分钟）

1. 把当日新增想法写入 `01-论文生产/选题管理/00-选题记录.md`
2. 同步章节进度到 `rules/控制面板.md`
3. 若章节结构发生变化，更新对应方法论文件
4. 新增 PDF 文献时，按 `ljg-xray-paper` 抽取为 Markdown 并存入 `docs/论文Markdown库/`

### 每周

1. 清点 `HUST-PhD-Thesis-Latex/body/chapter/` 的 TODO
2. 补齐一个"最卡脖子"的模块（证明/实验/图表三选一）
3. 将本周关键结论沉淀到 `docs/memory/`

### 每月

1. 评估目录是否仍符合你的"第一反应"
2. 清理长期未使用模板，避免系统臃肿
3. 回顾投稿路径并更新投稿记录

## 生命周期管理

选题生命周期：`00-选题记录.md` → `01-待深化/` → `02-待投稿/` → `03-已投稿/`

章节生命周期：草稿完成 → 理论证明补齐 → 实验验证补齐 → 终稿润色
