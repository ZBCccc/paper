# 工作流程

## 论文工程协作流程（当前核心）

1. 在 `HUST-PhD-Thesis-Latex/body/chapter/` 推进章节正文
2. 把结构性结论回写到 `docs/` 对应文件
3. 把关键研究记忆沉淀到 `docs/memory/`
4. 在 `rules/控制面板.md` 维护章节状态与近期任务

## 参考资料管理

- **核心论文库**：记录定义性与奠基性论文
- **相关工作库**：按技术路线梳理进展与差异
- **基础库**：维护术语、模型、原语、假设等底层知识
- **论文 Markdown 库**：将 `ref-thesis/` 中 PDF 抽取为 Xray 结构化 `.md`（默认 `ljg-xray-paper`），统一存放在 `docs/论文Markdown库/`

## PDF 文献处理规则

1. 新增 PDF 后，优先按 `ljg-xray-paper` 抽取为 Xray Markdown
2. Markdown 文件统一放入 `docs/论文Markdown库/`
3. 文件名使用 `作者-年份-标题关键词.md` 规范
4. 抽取后在文档头部记录来源 PDF 路径与 DOI
5. 需要保留全文转储时，另存为 `*.raw.md`，不作为主阅读材料

## 与 AI 交互规范

1. 每次开工先说明你在写哪一章、卡在哪一段
2. 明确输出类型：改写、补证、补实验、查引用、结构优化
3. 让 AI 同步更新对应系统文件，而不仅是单次回答
4. 完成后回写进度到 `rules/控制面板.md` 或 `docs/memory/`


章节生命周期：草稿完成 → 理论证明补齐 → 实验验证补齐 → 终稿润色
