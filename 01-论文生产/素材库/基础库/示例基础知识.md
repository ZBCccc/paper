# 基础知识速记卡（示例）

## 1. 资格检验（Eligibility Check）

- **定义**：在多关键词搜索中，判断候选文档是否满足剩余关键词约束的过程
- **在本文中的作用**：决定结果集合是否正确、是否完整
- **风险点**：若仅依赖概率结构，服务器行为难以被用户独立核验

## 2. Bloom Filter

- **优点**：空间效率高、查询快
- **缺点**：存在假阳性，不提供可验证证据链
- **对本文启示**：适合“过滤”，不适合“可追责验证”

## 3. Merkle Hash Tree (MHT)

- **核心能力**：根哈希作为全局承诺，认证路径支持成员性/非成员性证明
- **在本文中的作用**：为资格检验结果提供可验证证据
- **代价**：更新阶段需要维护树结构与认证信息

## 4. 承诺机制（Commitment）

- **绑定性**：承诺后不可任意改值
- **隐藏性**：承诺本身不泄露明文
- **在本文中的作用**：约束服务器无法伪造与真实结果不一致的校验信息

## 5. 泄露函数（Leakage Function）

- **定义**：SSE 安全模型中允许对手观察到的信息边界
- **在本文中的要求**：可验证增强不应显著扩大原有泄露范围
